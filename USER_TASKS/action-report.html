1 Connect GitHub to Vercel for Autodeploy Required

Right now, deployments happen manually via CLI. This connects them so every git push to main auto-deploys your site.

Open Vercel Git Settings

  1. Open your browser and go to:
    https://vercel.com/raposoignacio-2477s-projects/latino-events/settings/git
  2. If you're not logged in, log in with the same account you used for the CLI (the one tied to raposo.ignacio@gmail.com).
  3. You should see a page titled "Git" under your project settings.

Connect the Repository

  1. On that page, find the "Connected Git Repository" section. It should say "No repository connected" or similar.
  2. Click the "Connect Git Repository" button.
  3. A popup will appear asking you to select a Git provider. Click "GitHub".
  4. If this is your first time connecting GitHub to Vercel, a new tab/popup will open asking you to authorize the Vercel GitHub App. Click "Install" or "Authorize".
  5. You may be asked which repositories to grant access to. Select "Only select repositories" and pick latino-events-main. Then click "Install".
  6. Back in the Vercel popup, you should now see your repos listed. Find Ithelastghostl/latino-events-main and click on it.
  7. It will ask you to confirm the production branch. Make sure it says main. If it doesn't, change it to main.
  8. Click "Connect".

Verify It Works

  1. The page should now show "Connected to Ithelastghostl/latino-events-main" with branch main.
  2. To test: make any tiny change in the repo (even edit a comment), commit, and git push. Then check the Vercel dashboard — you should see a new deployment appear automatically within 30 seconds.

2 Configure Supabase Auth (Magic Link Login) Required

The past-events page lets users sign in with a magic link email to see event photos. Supabase Auth handles this, but you need to tell it which URLs are allowed.

Set the Site URL

  1. Open your browser and go to:
    https://supabase.com/dashboard/project/iialbhtgugmkmbrhwlud/auth/url-configuration
  2. Log in if prompted (use the account you created the Supabase project with).
  3. You'll see a page titled "URL Configuration".
  4. In the "Site URL" field at the top, delete whatever is there and type:
    https://www.latribu.co.uk
  5. Click "Save".

Add Redirect URLs

  1. On the same page, scroll down to the "Redirect URLs" section.
  2. Click "Add URL".
  3. Type: https://www.latribu.co.uk/** — then click "Add URL" to confirm.
  4. Click "Add URL" again.
  5. Type: https://latino-events.vercel.app/** — then click "Add URL" to confirm.
  6. Click "Add URL" one more time.
  7. Type: http://localhost:*/** — then click "Add URL" to confirm.
  8. You should now see all three URLs listed. Click "Save" if there's a save button, or they may save automatically.
The /** wildcard means "any path on this domain". The localhost entry is for local development/testing.

Verify Email Auth Is Enabled

  1. In the left sidebar, click "Authentication", then click "Providers" (or go directly to:
    https://supabase.com/dashboard/project/iialbhtgugmkmbrhwlud/auth/providers)
  2. You'll see a list of auth providers (Email, Phone, Google, etc.).
  3. Click on "Email" to expand it.
  4. Make sure the toggle at the top says "Enabled" (green). If it's off, click it to turn it on.
  5. Make sure "Confirm email" is toggled ON. This is what sends the magic link.
  6. Make sure "Secure email change" is toggled ON.
  7. Leave everything else as default.
  8. Click "Save".

4 Replace Fatsoma Placeholder URLs Required

All "Get Your Tickets" buttons currently link to a placeholder. You need to swap in your real Fatsoma event URLs.

How to get your Fatsoma event URLs

  1. Go to fatsoma.com and log in to your promoter account.
  2. Go to your event page. The URL in your browser bar is what you need. It looks like:
    https://www.fatsoma.com/e/abc123/your-event-name
  3. Copy this URL.

How to replace the placeholders

  1. Open index.html in your code editor (VS Code, Sublime, etc.).
  2. Press Ctrl+H (Windows/Linux) or Cmd+H (Mac) to open Find and Replace.
  3. In the "Find" field, paste: https://www.fatsoma.com/e/placeholder
  4. In the "Replace" field, paste your real Fatsoma URL.
  5. Click "Replace All" if all events go to the same link. If each event card has a different URL, click "Replace" one at a time and paste the correct URL for each.
  6. Save the file (Ctrl+S / Cmd+S).
  7. Repeat for any other HTML files that have ticket links (promotions.html, etc.).

To find all files with placeholder links (terminal)

cd ~/projects/latino-events
grep -rn "fatsoma.com/e/placeholder" *.html

This shows every file and line number where the placeholder appears.

After replacing: push to GitHub

git add *.html

6 Set Up Resend Email Marketing Required

Email collection already works — when someone enters their email on the site, it's saved to Supabase. Resend is the service that will send emails to those subscribers (welcome emails, event announcements, promos). Free tier: 3,000 emails/month.

Step 6.1 — Create a Resend account

  1. Open your browser and go to: https://resend.com/signup
  2. Click "Sign up with GitHub" (easiest since you already have a GitHub account) — or use email/Google.
  3. If you chose GitHub: a popup will ask you to authorize Resend. Click "Authorize resend".
  4. You'll land on the Resend dashboard. You now have an account.

Step 6.2 — Add your domain to Resend

  1. In the Resend dashboard, look at the left sidebar. Click "Domains".
  2. Click the "Add Domain" button (top right).
  3. In the text field, type: latribu.co.uk
  4. For "Region", leave the default (usually "US East (N. Virginia) / us-east-1") unless you have a reason to change it.
  5. Click "Add".
  6. Resend will now show you a page titled something like "Verify latribu.co.uk". This page lists DNS records you need to add. Do NOT close this page. Keep it open — you'll need it for the next step.
  7. You'll see a table with rows like this (the exact values will be different — copy from YOUR screen, not from this guide):
TypeName / HostValuePriority
TXTresend._domainkey(a long string starting with p=)—
TXT@v=spf1 include:amazonses.com ~all—
MXbouncefeedback-smtp.us-east-1.amazonses.com10
The MX record is optional — it's only needed if you want to handle bounce/reply emails. The two TXT records are required.

Step 6.3 — Add the DNS records in Vercel (your DNS is managed here, not GoDaddy)

Why Vercel and not GoDaddy? Your domain latribu.co.uk is registered at GoDaddy, but the nameservers point to Vercel. This means Vercel controls all DNS records. GoDaddy will show "DNS is managed elsewhere" and won't let you edit records there. All DNS changes go through the Vercel dashboard.

Keep the Resend page open in one tab. Open Vercel in a new tab.

  1. Go to: https://vercel.com/raposoignacio-2477s-projects/latino-events/settings/domains
  2. If you're not logged in, log in with your Vercel account.
  3. You should see your domain(s) listed (e.g., latribu.co.uk or www.latribu.co.uk).
  4. Click on latribu.co.uk (the root domain, without www). This takes you to the domain's DNS management page.
  5. If that doesn't take you to DNS records, try going directly to: https://vercel.com/teams/raposoignacio-2477s-projects/domains/latribu.co.uk
  6. You should see a page with a list of DNS records and an "Add Record" button or form at the top.

Add the first TXT record (DKIM):

  1. At the top of the DNS records page, you'll see a form with dropdowns/fields. Set the "Type" dropdown to TXT.
  2. In the "Name" field, type: resend._domainkey
  3. In the "Value" field, go to the Resend tab in your browser. Find the first row in the DNS records table (the DKIM one). It has a long value starting with something like p=MIGfMA0GCS.... Click the copy icon next to it (or manually select and copy the entire value).
  4. Go back to the Vercel tab and paste that value into the "Value" field.
  5. Click "Add".
  6. The record should appear in the list below.

Add the second TXT record (SPF):

  1. Set the "Type" dropdown to TXT again.
  2. In the "Name" field, type: @
  3. In the "Value" field, type exactly: v=spf1 include:amazonses.com ~all
  4. Click "Add".
Important: Look at the existing DNS records in the list. If you already see a TXT record with Name @ that starts with v=spf1, do NOT add a new one. Instead, click the three dots (…) or edit icon next to the existing record, and add include:amazonses.com before the ~all. For example, change v=spf1 include:vercel.com ~all to v=spf1 include:vercel.com include:amazonses.com ~all. Then click "Save". You can only have ONE SPF record per domain — having two will cause email delivery failures.

Add the MX record (optional, for bounce handling):

  1. Set the "Type" dropdown to MX.
  2. In the "Name" field, type: bounce
  3. In the "Value" field, type: feedback-smtp.us-east-1.amazonses.com
  4. In the "Priority" field (should appear for MX records), type: 10
  5. Click "Add".
The MX record is optional. It handles bounce notifications (emails that couldn't be delivered). You can skip it for now and add it later.

Step 6.4 — Verify the domain in Resend

  1. Go back to the Resend tab (the "Verify latribu.co.uk" page).
  2. Click the "Verify DNS Records" button.
  3. If DNS has propagated, each record will show a green checkmark. This can take anywhere from 1 minute to 48 hours, but usually completes within 5–10 minutes.
  4. If some records still show as pending, wait a few minutes and click "Verify" again.
  5. Once all records are green, the domain status will change to "Verified". You can now send emails from any address @latribu.co.uk.

Step 6.5 — Create a Resend API key

  1. In the Resend left sidebar, click "API Keys".
  2. Click "Create API Key" (top right).
  3. In the form that appears:
    • Name: type la-tribu-production
    • Permission: select "Sending access" (NOT "Full access" — we only need to send emails)
    • Domain: select latribu.co.uk from the dropdown
  4. Click "Create".
  5. A popup will show your API key. It looks like: re_AbCdEfGh_1234567890abcdef
  6. Copy this key immediately and save it somewhere safe (a password manager, a secure note, etc.). Resend will never show it again.

Step 6.6 — Store the API key in Supabase as a secret

The API key must be stored as a "secret" in Supabase so the Edge Function can use it without the key being visible in your code.

Option A — Via the Supabase Dashboard (easiest):

  1. Go to: https://supabase.com/dashboard/project/iialbhtgugmkmbrhwlud/settings/functions
  2. Scroll down to "Edge Function Secrets" (or look for a section called "Secrets").
  3. Click "Add new secret".
  4. For "Name", type exactly: RESEND_API_KEY
  5. For "Value", paste the API key you copied in Step 6.5 (the re_... string).
  6. Click "Save".

Option B — Via the terminal:

  1. Open your terminal.
  2. Install the Supabase CLI if you don't have it:
    npm install -g supabase
  3. Link to your project:
    supabase link --project-ref iialbhtgugmkmbrhwlud
  4. It will ask for your database password. Enter the password you set when creating the Supabase project. If you forgot it, you can reset it in the Supabase dashboard under Settings → Database.
  5. Set the secret:
    supabase secrets set RESEND_API_KEY=re_YOUR_KEY_HERE
    (Replace re_YOUR_KEY_HERE with your actual key.)

  2. Claude will create and deploy a Supabase Edge Function that:

8 Add Past Events to the Database When ready

The past-events page pulls data from Supabase. Right now the database is empty, so the page will show no events. Here's how to add them.

Add an event via the Supabase Dashboard

  1. Go to: https://supabase.com/dashboard/project/iialbhtgugmkmbrhwlud/editor
  2. In the left sidebar, click on the table past_events.
  3. Click the "Insert row" button (top right, green button with a "+" icon).
  4. Fill in the fields:
    • id: leave blank (auto-generated)
    • title: e.g., Noche de Salsa Vol. 1
    • description: e.g., Our first ever event — 200 people dancing until 3am at The Lighthouse in Shoreditch.
    • date: e.g., 2026-03-15 (YYYY-MM-DD format)
    • venue: e.g., The Lighthouse, Shoreditch
    • cover_image_url: a URL to the event's cover photo (you can upload images to Supabase Storage or use any image hosting). Leave blank if you don't have one yet.
    • created_at: leave blank (auto-generated)
  5. Click "Save" (or the checkmark button).
  6. The event will now appear on the past-events page of your site.

Add photos for an event

  1. First, you need the event ID. In the table editor, find the event you just added. The id column shows a long string like a1b2c3d4-e5f6-7890-abcd-ef1234567890. Copy it.
  2. In the left sidebar, click on past_event_photos.
  3. Click "Insert row".
  4. Fill in:
    • id: leave blank
    • event_id: paste the event ID you copied
    • image_url: URL of the photo
    • caption: optional description of the photo
    • created_at: leave blank
  5. Click "Save".
  6. Repeat for each photo in that event.
Photos are only visible to signed-in users. When someone visits past-events.html and signs in with their email (magic link), the photos will appear.
