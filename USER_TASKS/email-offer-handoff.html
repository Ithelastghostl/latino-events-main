How to send a one-off promotion / newsletter

No send button was built into the site (by design — no admin panel). Send promos from the Resend dashboard:

  1. Get your subscriber emails: Supabase → Table Editor → subscribers → export CSV.
  2. Resend → Audiences → import that CSV.
  3. Resend → Broadcasts → write & send to that audience, from hola@latribu.co.uk.

A reusable branded promo template (promoEmail()) lives in supabase/functions/_shared/emails.ts if you later want to automate promos — but that would be a new build.

What I cleaned up (and why)

An earlier welcome-email system existed: a database trigger (on_new_subscriber → notify_new_subscriber) that called the old send-welcome-email function on every signup. With the new system also sending a welcome, every signup was firing two emails — and the old one had no offer code and used the wrong sender (hello@ not hola@).

With your approval I removed the trigger and its function (migration remove_legacy_welcome_email_trigger). Now each signup sends exactly one correct email. The old edge function is left orphaned for you to delete (TODO #2). Restore SQL, if ever needed:

-- (the dropped function bodied a webhook to send-welcome-email) CREATE TRIGGER on_new_subscriber AFTER INSERT ON public.subscribers FOR EACH ROW EXECUTE FUNCTION notify_new_subscriber();

Security notes (informational)

  • Good: The Resend API key lives only as a Supabase secret, never in the website code. The site only holds the public anon key (correct).
  • Good: contact_messages has row-level security on with no public read policy — only the server can read enquiries.
  • Done — hardened: The pre-existing anon_insert_subscribers rule (which let anyone insert rows into subscribers directly via the public API) has been dropped (migration drop_anon_insert_subscribers_policy). With signups now flowing only through the secure subscribe function (service-role), the table now has no anon policies — RLS default-deny. Verified: a direct anon insert is rejected with 42501 (RLS violation), while the subscribe function still works. Spam-insert vector closed.
